# Requisitos Não Funcionais

## Introdução

Esta página reúne todos os requisitos não funcionais elicitados por meio da técnica de entrevista para o sistema da **Agência Korus**.

## Metodologia

Os requisitos foram organizados em tabelas agrupadas por categoria de qualidade do sistema. Cada requisito possui um identificador único (**ID**) e sua respectiva descrição.

**Legenda:**

- **RNFxx** — Requisito Não Funcional nº xx

---

## 1. Usabilidade

A Tabela 1 apresenta os requisitos não funcionais relacionados à usabilidade do sistema.

<center><b>Tabela 1</b> — Requisitos Não Funcionais de Usabilidade</center>

| ID    | Descrição |
| ----- | --------- |
| RNF01 | O sistema deve possuir interface intuitiva e de fácil navegação para usuários sem conhecimento técnico. |
| RNF02 | A landing page deve apresentar design moderno, atrativo e alinhado à identidade visual da Agência Korus. |
| RNF03 | O sistema deve ser responsivo, funcionando adequadamente em desktop, tablet e smartphone. |
| RNF04 | O sistema deve priorizar boa experiência do usuário na navegação da página inicial e no preenchimento de formulários. |
| RNF05 | O sistema deve organizar as informações de forma clara, evitando excesso de etapas para contato ou solicitação de orçamento. |
| RNF06 | As páginas devem manter consistência visual quanto a cores, tipografia, botões e componentes. |

---

## 2. Desempenho

A Tabela 2 apresenta os requisitos não funcionais relacionados ao desempenho do sistema.

<center><b>Tabela 2</b> — Requisitos Não Funcionais de Desempenho</center>

| ID    | Descrição |
| ----- | --------- |
| RNF07 | A plataforma deve apresentar tempo de carregamento adequado, especialmente na landing page. |
| RNF08 | O sistema deve suportar múltiplos acessos simultâneos sem comprometer a experiência do usuário. |
| RNF09 | O envio de formulários deve ocorrer com tempo de resposta aceitável. |
| RNF10 | O sistema deve tratar adequadamente o upload e a exibição de mídias como imagens e vídeos no portfólio. |

## 3. Segurança

A Tabela 3 apresenta os requisitos não funcionais relacionados à segurança do sistema.

<center><b>Tabela 3</b> — Requisitos Não Funcionais de Segurança</center>

| ID    | Descrição |
| ----- | --------- |
| RNF11 | O sistema deve garantir autenticação segura dos usuários. |
| RNF12 | As senhas devem ser armazenadas de forma criptografada. |
| RNF13 | O sistema deve proteger dados pessoais e dados comerciais dos clientes. |
| RNF14 | O sistema deve aplicar controle de acesso por perfil e permissão. |
| RNF15 | O sistema deve impedir que clientes acessem projetos de outros clientes. |
| RNF16 | O sistema deve impedir que funcionários acessem funções exclusivas de administradores. |
| RNF17 | O sistema deve utilizar comunicação segura, preferencialmente com HTTPS. |
| RNF18 | O sistema deve possuir mecanismos de proteção contra acessos indevidos, injeção de dados maliciosos e falhas comuns em aplicações web. |
| RNF19 | O sistema deve respeitar boas práticas de privacidade e proteção de dados, especialmente em formulários de captação de leads. |
